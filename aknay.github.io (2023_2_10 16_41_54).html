概述

在本教程中，我们将讨论 MVVM 以及如何在 Flutter 应用程序中将业务逻辑与 UI 分离。为此，我们将讨论 Sink、Steam 和 Steam Controller。最后，我们将把所有这些想法应用到一个非常简单的计数器程序中。

处理状态

Flutter 中的一切都是反应式的，这意味着您可以观察每个用户 UI 操作并对其采取行动。为了观察用户操作，Flutter 使用与 React 相同的模式，称为Lifting State Up。您可以观看精彩的演讲来了解 Flutter 中的状态。从谈话中，希望你明白为什么我们需要将 UI 和逻辑分开。我们不想将 UI 和逻辑混合在一起，不是吗？让我们看看下一节如何使用 Flutter Stream 来分离 UI 和业务逻辑。

接收器、流和流控制器

我们需要在 Flutter 的异步编程中做，才能得到一个数据序列。我们需要 3 个部分；接收器、流和流控制器。在 Flutter 中，Sink 基本上是您将输入数据传递给它的地方。它也被视为数据消费者。流是一个数据生成器，您可以从中获取输出。流控制器作用于接收器和流之间。控制器操纵来自接收器的数据并将操纵的数据传递到流。它是一种从数据到数据输出的单向通信方式。为了分离 UI 和业务逻辑，我们将在 UI 处暴露 Sink 和 Stream，并将业务逻辑与 Steam Controller 一起隐藏在一个单独的文件中。
