作者

因此，我要声明我不是 Axum 专家，并且可能有方法调整 State 以达到同样的目的，我的理解是，State 虽然比 Extensions 更类型安全，但必须保证并适用于任何给定的路由器，而 Extensions 可以为一部分路由动态加载。

因此，要在具有可变用户角色的多租户系统中使用 State 进行身份验证/授权，您需要为每种类型的用户设置单独的路由器，以便在 State 中加载角色范围的用户。

相反，我使用“管理员”中间件来控制管理路由，该中间件基于“员工”中间件和“会话用户”中间件（对于登录用户而言，不一定是任何租户组织的成员）。因此，管理员控制的路由可以期望ApplicationState具有数据库连接池的公共对象以及OrgAdmin具有授权用户详细信息的扩展。

请参阅下面的示例路由器来演示我的意思：

  let api_organization = Router::new()
    // UPDATE a tenant organization details, requires a user with an ADMIN membership within the tenant org
    .route("/", put(routes_v1::org::update))
    .route_layer(axum::middleware::from_fn(middleware::admin::auth))
    // GET a tenant org details, requires a user with a valid membership within the tenant org
    .route("/", get(routes_v1::org::details))
    .route_layer(axum::middleware::from_fn_with_state(
      state.clone(),
      middleware::employee::auth,
    ))
    // CREATE a tenant org, requires a session user without any specific org membership
    .route("/", post(routes_v1::org::create))
    .route_layer(axum::middleware::from_fn_with_state(
      state.clone(),
      middleware::session_user::auth,
    ))

如果有更好/更常规的方法可以做到这一点，我会非常高兴。@davidpdrsn你对于这种模式有什么想法吗？
