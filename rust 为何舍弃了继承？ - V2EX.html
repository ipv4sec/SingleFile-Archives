    Go 也舍弃了继承。继承缺点很多，而优点几乎都可以被类似 mix-in 的模式取代，因此曾经一段时间大家提倡少用继承，并且 Java 之类原本很强调继承的语言都增加了对 mix-in 的支持。

    extends 继承，是父子关系。子类拥有父类的功能。达到复用。父类比较大，子类弄些不需要的功能没有意义。

    trait 类似于兄弟关系。两个平等的组件需要什么功能加什么功能就得了。可以做到更加合理一点。变向也能达到父子的功能。
