samba使用ldap认证

samba ldap

Samba使用LDAP认证可以提供更加灵活的用户管理和访问控制，下面是具体的配置方法：

1. 安装LDAP服务器和客户端

首先需要安装OpenLDAP服务器和客户端：


# yum install openldap-servers openldap-clients

2. 创建LDAP目录结构

使用ldapadd命令创建LDAP目录结构，可以使用类似以下内容的LDIF文件：


# base.ldif
dn: dc=mydomain,dc=com
objectClass: top
objectClass: dcObject
objectClass: organization
o: mydomain.com
dc: mydomain
dn: cn=Manager,dc=mydomain,dc=com
objectClass: organizationalRole
cn: Manager
userPassword: secret

执行以下命令创建LDAP目录结构：


# ldapadd -x -D "cn=Manager,dc=mydomain,dc=com" -w secret -f base.ldif

3. 添加Samba相关属性

需要添加samba.schema文件，该文件包含了Samba需要用到的属性定义，可以从Samba源码包中找到。

将samba.schema文件复制到OpenLDAP的schema目录：


# cp /usr/share/doc/samba-<version>/LDAP/samba.schema /etc/openldap/schema/samba.schema

然后在slapd.conf文件中添加以下内容：


include         /etc/openldap/schema/samba.schema

其中，slapd.conf文件位置可能是不同的，需要根据具体情况进行修改。

添加完毕后，需要重启LDAP服务器以使配置生效：


# systemctl restart slapd

4. 创建Samba账号

使用ldapadd命令创建Samba账号，可以使用类似以下内容的LDIF文件：


#sambauser.ldif
dn: uid=alice,ou=People,dc=mydomain,dc=com
objectClass: top
objectClass: person
objectClass: organizationalperson
objectClass: inetorgperson
objectClass: posixAccount
objectClass: shadowAccount
objectClass: sambaSamAccount
uid: alice
cn: Alice Testuser
gidNumber: 1000
homeDirectory: /home/alice
uidNumber: 10000
loginShell: /bin/bash
userPassword: {crypt}x
shadowLastChange: 0
shadowMax: 0
shadowWarning: 0
sambaSID: S-1-5-21-3791562307-893885483-826636955-30000
sambaLMPassword: 0CB694880250E2271B8E9C0638DEE3C3
sambaNTPassword: F5FFFB9D51D58539AAD3B435B51404EE
sambaPrimaryGroupSID: S-1-5-21-3791562307-893885483-826636955-1000
sambaAcctFlags: [U]
sambaPwdLastSet: 0
sambaHomeDrive: H:
sambaProfilePath: \\mydomain.com\profiles\%u
sambaLogonScript: logon.bat
sambaLogonTime: 0
sambaLogoffTime: 2147483647
sambaKickoffTime: 2147483647
sambaPwdCanChange: 0
sambaPwdMustChange: 2147483647

执行以下命令创建Samba账号：


# ldapadd -x -D "cn=Manager,dc=mydomain,dc=com" -w secret -f sambauser.ldif

5. Samba配置

修改Samba配置文件，可以使用如下内容：


[global]
workgroup = MYDOMAIN
server string = Samba Server Version %v
ldap suffix = dc=mydomain,dc=com
ldap user suffix = ou=People
ldap group suffix = ou=Groups
ldap machine suffix = ou=Computers
ldap idmap suffix = ou=Idmap
ldap ssl = off
passdb backend = ldapsam:ldap://localhost
ldap admin dn = cn=Manager,dc=mydomain,dc=com
ldap passwd sync = yes
ldap delete dn = yes
auth methods = sam_ldap
[homes]
comment = Home Directories
valid users = %S
read only = No
browseable = No
[printers]
comment = All Printers
path = /var/spool/samba
printable = Yes
browseable = No

其中，passdb backend指定使用LDAP进行身份认证，ldap服务器地址为localhost，ldap管理员dn为cn=Manager,dc=mydomain,dc=com。auth methods指定使用sam_ldap进行身份认证。

6. 测试

重启Samba服务，然后使用Samba用户名和密码进行测试：


# systemctl restart smb
# smbclient -U alice //localhost/homes

如果能够成功登录，则表明Samba使用LDAP认证配置成功。
